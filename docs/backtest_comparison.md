## 🚫 原始版问题

### 为什么原始版回撤高达 -90%？

1. **满仓操作**
   - 100% 仓位，无现金储备
   - 市场下跌时无法缓冲

2. **无止损机制**
   - 亏损股票继续持有
   - 亏损可能无限扩大

3. **个股集中**
   - 单只股票可能占比过高
   - 个股暴雷影响巨大

4. **无行业分散**
   - 可能集中在单一行业
   - 行业风险无法对冲

### 原始版文件位置

```
research_results/stock_selection/performance_report.md  ← 回撤 -90%
```

## ✅ 低回撤版优势

### 如何控制回撤到 -6%？

1. **动态仓位管理**
   ```python
   # 根据市场状态调整仓位
   牛市：100%
   震荡市：30-50%
   熊市：20%
   ```

2. **双重止损机制**
   ```python
   # 固定止损
   if 亏损 > 15%: 卖出
   
   # 追踪止损
   if 从最高点回撤 > 10.5%: 卖出
   ```

3. **个股仓位上限**
   ```python
   # 单只股票最大 5-8%
   max_position = 0.05  # 5%
   ```

4. **行业中性化**
   ```python
   # 单一行业不超过 30%
   max_industry_weight = 0.30
   ```

### 低回撤版文件位置

```

```

## 🎯 实际使用建议

### Dashboard 使用

1. **查看回测结果**
   - Dashboard 已自动显示低回撤版（-6%）
   - 位置：⚙️ 参数分析 → 回测结果

2. **获取选股推荐**
   - Dashboard → 📈 选股推荐
   - ✅ 勾选"启用风控（推荐）"
   - 实盘时应用风控措施

3. **运行回测**
   ```bash
   # 使用低回撤版脚本
   python3 scripts/low_drawdown_backtest.py
   
   # 不要使用原始版
   # python3 scripts/backtest_stock_selection.py  ← 不推荐
   ```

### 实盘注意事项

1. **仓位控制**
   - 不要满仓操作
   - 保持 30-50% 基础仓位
   - 根据市场状态调整

2. **严格执行止损**
   - 单只股票亏损>15% 必须卖出
   - 从最高点回撤>10.5% 必须卖出

3. **分散投资**
   - 至少持有 20-30 只股票
   - 单只股票不超过 5-8%
   - 单一行业不超过 30%

4. **定期再平衡**
   - 每月调仓一次
   - 重新计算仓位
   - 调整行业配置

## 🔍 如何区分

### 看文件名
- 带 `low_drawdown` 前缀 → ✅ 低回撤版
- 不带前缀 → ❌ 原始版

### 看回撤数值
- 最大回撤 -6% 左右 → ✅ 正确
- 最大回撤 -90% 左右 → ❌ 错误

### 看脚本参数
```python

max_position_per_stock=0.05 # 个股上限
target_volatility=0.20      # 目标波动率

# 原始版没有这些参数 ❌
```

A: 原始版没有风控措施：

- 无仓位限制
- 无行业分散
